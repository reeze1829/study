#  C++ 문법 학습 정리

C++의 핵심 문법 정리!

##  학습 목차

### 1. 기초 문법 (Basics)
* **1-1. iostream & Namespace**
  - `cin`, `cout` 활용 및 `std::` 명시적 사용의 중요성 학습
* **1-2. 함수 오버로딩**
  - 같은 이름의 함수를 매개변수(타입, 개수, 순서)에 따라 다르게 정의하는 방법 학습4
* **1-3. 디폴트 매개변수**
  - 인자 생략 시 자동 적용되는 기본값 설정 및 호출 규칙 학습
* **1-4. 인라인 함수**
  - 매크로 함수를 대체하는 `inline` 키워드와 함수 호출 오버헤드 최적화 학습
### 2. 참조와 메모리 (References & Memory)
* **2-1. 참조자 (Reference)**
  - 변수에 붙이는 별명(Alias) 개념과 포인터와의 차이점 학습
* **2-2. 참조에 의한 호출 (Call-by-Reference)**
  - 함수 매개변수로 참조자를 전달하여 복사 없이 원본을 수정하는 최적화 기법 학습
* **2-3. new & delete (동적 할당)**
  - C의 malloc/free를 대체하는 연산자와 힙(Heap) 메모리 관리 방법 학습
* **2-4. const 데이터 보호 전략**
  - 포인터와 참조자에 const를 결합하여 프로그램 안정성을 높이는 방법 학습
### 3. 객체지향 프로그래밍 기초 (OOP Basics)
* **3-1. 클래스와 객체 (Class & Object)**
  - 설계도(Class)와 실체(Object)의 관계 및 인스턴스의 독립적 메모리 구조 학습
* **3-2. 접근 제어자와 정보 은닉 (Access Modifiers)**
  - `public`, `private`, `protected`를 이용한 보안 설정 및 Getter/Setter를 통한 유지보수(유연성) 전략 학습
* **3-3. 생성자, 소멸자 및 초기화 리스트**
  - 객체의 생명주기 관리 및 초기화 리스트를 통한 성능 최적화와 RAII 패턴 학습
* **3-4. 정적 멤버 (Static Member)**
  - 모든 객체가 공유하는 공용 변수의 개념과 객체 없이 호출 가능한 정적 함수의 특성 학습
### 4. 객체지향 프로그래밍 심화 
* **4-1. 상속 (Inheritance)**
  - 기존 클래스(Person)의 특성을 유지하며 새로운 기능(Student)을 확장하는 계층 구조 학습
* **4-2. 다형성 (Polymorphism)**
  - `virtual` 키워드를 이용한 동적 바인딩과 가상 함수 테이블(vtable)의 원리 학습
  - 추상 클래스와 순수 가상 함수를 활용한 객체지향 설계 표준(Interface) 정립
* **4-3. abstract-class (추상 클래스)**
  - 순수 가상 함수를 활용한 인터페이스 설계 및 표준화 규격 학습
  - 다형성을 통한 이종 객체(TV, 에어컨)의 통합 관리 방식 이해
* **4-4. operator-overloading (연산자 오버로딩)**
  - 기본 연산자(+, ==, << 등)를 객체에 맞게 재정의하여 직관적인 코드 설계 학습
  - `friend` 키워드와 `*this` 반환을 이용한 연산자 구현 및 메모리 관리 이해
### 5. 표준 템플릿 라이브러리 (STL Basics)
* **5-1. template-function (함수 템플릿)**
  - 자료형을 파라미터화하여 코드 중복을 제거하는 일반화 프로그래밍 학습
  - 컴파일 타임의 인스턴스화 원리 및 포인터 타입 사용 시 주의사항 이해
* **5-2. vector-container (동적 배열)**
  - 크기가 유동적으로 변하는 `std::vector`의 인터페이스 및 내부 동작 원리 학습
* **5-3. string-class (표준 문자열)**
  - C언어 `char*`의 한계를 극복한 `std::string`의 자동 메모리 관리 이해
  - 문자열 검색(`find`), 추출(`substr`), C 스타일 변환(`c_str`) 실습
* **5-4. iterator (반복자)**
  - 컨테이너 내부 원소를 가리키는 '일반화된 포인터'의 개념 이해
  - `begin()`과 `end()`를 이용한 반개방 범위(`[first, last)`) 순회 표준 정립
  - `*`(역참조)와 `++`(이동) 연산자를 활용한 데이터 접근 및 수정 실습
  - `auto` 키워드를 이용한 복잡한 반복자 타입 선언 최적화 및 `reverse_iterator` 활용
  
##  주요 포인트 요약

### 1-1. iostream & Namespace
- **std::cin/cout**: C++의 표준 입출력. `<<`, `>>` 연산자 방향 주의.
- **Namespace std**: 이름 충돌 방지를 위해 실무에서는 `using namespace std;` 보다는 `std::`를 직접 붙이는 것을 권장함.
### 1-2. 함수 오버로딩
- **핵심 원칙**: 기준은 무조건 **매개변수** (타입, 개수, 순서)
- **주의사항**: 매개변수가 같고 반환 타입만 다르면 컴파일 에러 발생
### 1-3. 디폴트 매개변수
- **우선순위 규칙**: "값 전달은 왼쪽부터, 설정은 오른쪽부터!" (기본값은 반드시 오른쪽 끝부터 채워야 함)
- **건너뛰기 불가**: C++은 특정 인자를 건너뛰고 다음 인자에 값을 넣는 기능을 지원하지 않음.
- **모호성 주의**: 호출 형태가 오버로딩된 다른 함수와 겹칠 경우 컴파일 에러 발생.
### 1-4. 인라인 함수
- **핵심 원리**: 함수 호출 위치에 함수의 몸체 코드를 직접 복사해 넣어 '점프(Call)' 없이 실행함.
- **매크로와의 차이**: 매크로는 단순 글자 바꾸기라 연산 우선순위 오류가 잦지만, 인라인은 인자를 '값'으로 먼저 계산한 뒤 확장하므로 안전함.
- **성능 최적화**: 짧은 함수에 사용하여 실행 속도를 높이나, 너무 긴 함수를 인라인화하면 프로그램 용량이 커질 수 있음.
### 2-1. 참조자
- **개념**: 기존 변수에 붙이는 '별명'. 새로운 메모리를 할당하지 않고 원본 공간을 공유함.
- **특징**: `&` 기호 사용, 주소값이 원본과 동일하며 별명을 수정하면 원본도 바뀜.
- **3대 제약**: 선언 시 반드시 초기화 필수, NULL 불가, 한 번 정해진 대상은 변경(환승) 불가.
### 2-2. 참조에 의한 호출 (Call-by-Reference)
- **효율성**: 주소값을 직접 전달하는 포인터 방식과 성능은 같지만, 문법은 일반 변수처럼 간결함.
- **가독성**: 호출 시 `&`를 붙일 필요가 없고, 함수 내부에서도 `*` (역참조) 연산 없이 작업 가능.
- **const 참조자 **: `const T &` 형태로 사용하여 큰 데이터를 복사 비용 없이 전달하면서, 함수 내부에서 원본이 수정되는 것을 방지함
### 2-3. new & delete (동적 할당)
- **이점**: 형변환 불필요, `sizeof` 생략, 할당과 동시에 초기화 가능.
- **메모리 영역**:
  - **Stack**: 일반 변수 저장 영역. 함수 종료 시 자동 해제.
  - **Heap**: `new` 할당 공간. 수동 해제(`delete`) 필수, 미해제 시 **메모리 누수** 발생.
- **해제 규칙**: `new[]` 배열 할당 시 반드시 **`delete[]`** 사용. (메모리에 기록된 배열 길이를 참조하여 전체를 삭제하기 위함)
- **안전 습관**: 해제 후 포인터를 `nullptr`로 초기화하여 잘못된 메모리 접근 방지.
### 2-4. const 전략
- **공식**: `const T*`는 **값** 보호(내용물 상수화), `T* const`는 **방향** 보호(주소값 고정).
- **실무 표준**: `const T&` 매개변수로 **성능(복사 비용 X)**과 **안전(수정 불가)**을 동시에 확보.
- **의미**: `const`는 개발자 간의 **'데이터 신뢰'**를 보장하는 최소한의 안전장치이자 약속임.
### 3-1. 클래스와 객체
- **개념**: 설계도(Class)와 실체(Object) 관계 및 인스턴스 독립성 학습.
- **보안**: `class`는 보안 중심 설계(기본 private)라 외부 접근 시 `public:` 필수.
- **메모리**: 유연한 수명 관리를 위해 힙(`new`, `->`)과 스택(`.`) 사용법 구분.
### 3-2. 접근 제어자와 보안 설정
- **Public**: 누구나 사용 가능한 공식 통로.
- **Private**: 데이터 오염 방지를 위한 자물쇠. 내부 보조 함수도 숨겨서 사용 편의성 증대.
- **Protected**: 외부 접근은 금지하되, 상속 관계의 자식에게는 허용하는 중첩 보안.
- **Getter/Setter**: `private` 변수의 경호원 역할을 하며, 내부 로직 변경 시에도 외부 코드 수정을 최소화함(유지보수성/유연성).
### 3-3. 생성자, 소멸자 및 초기화 리스트
* **생명주기(Lifecycle)**: 객체 탄생(생성자)과 죽음(소멸자)의 자동 관리 및 역순 소멸 원리 이해.
* **초기화 리스트**: `const`와 참조자 멤버의 필수 초기화 방식이자, 대입보다 빠른 성능 최적화 기법.
* **RAII 패턴**: "자원 획득은 초기화와 동시에!" 생성자에서 할당하고 소멸자에서 해제하여 메모리 누수 방지.
### 3-4. 정적 멤버 (Static Member)
* **공유 변수(static)**: 모든 객체가 하나의 메모리 공간을 공유하며, 클래스 외부 초기화를 통해 Data 영역에 할당됨.
* **정적 함수(static)**: 객체 생성 없이 클래스명으로 호출 가능하며, 특정 객체에 종속되지 않아 일반 멤버 변수 접근은 불가함.
* **공유 증명**: 서로 다른 객체가 동일한 static 변수 주소를 가리키는 것을 실습으로 확인하여 공유 원리 정립.
* **생명주기**: 객체의 소멸과 상관없이 프로그램 시작부터 종료까지 메모리에 유지되는 특성 학습.
### 4-1. 상속 (Inheritance)
* **생성자 순서**: 자식 객체 생성 시 부모 생성자가 먼저 호출되며, 초기화 리스트를 통해 부모 생성자에 인자를 전달함.
* **접근 제어자**: `protected`는 외부 접근은 차단하되, 상속받은 자식 클래스에서의 직접 접근은 허용함.
* **상속의 3가지 방식**:
    * **public 상속**: 부모 멤버의 접근 권한을 그대로 유지 (IS-A 관계의 표준).
    * **protected 상속**: 부모의 public 멤버를 자식 내에서 protected로 하향 조정.
    * **private 상속**: 모든 부모 멤버를 자식 내에서 private으로 봉인하여 외부 접근 완전 차단.
### 4-2. 다형성 (Polymorphism)
* **상속과 다형성의 이유**: 
  - **상속**: 코드 중복을 방지하고 공통 속성을 부모에게 집중시켜 유지보수성 향상.
  - **다형성**: 부모 타입 포인터 하나로 여러 자식 객체를 일관되게 관리하는 확장성 확보.
* **가상 함수(virtual)**: 실행 시점(Runtime)에 실제 객체의 타입을 확인하여 함수를 호출하는 **동적 바인딩** 구현.
* **가상 소멸자**: 부모 포인터로 자식 객체 삭제 시, 자식 소멸자가 누락되지 않도록 하여 메모리 누수를 원천 봉쇄함.
* **순수 가상 함수(=0)**: 자식 클래스에게 특정 기능을 반드시 구현하도록 강제하는 '설계 규격'이며, 이를 포함한 클래스는 **추상 클래스**가 됨.
* **오버로딩 vs 오버라이딩**: 
  - **오버로딩**: 동일 클래스 내에서 매개변수를 다르게 하여 함수를 여러 개 정의.
  - **오버라이딩**: 상속 관계에서 부모 함수를 자식에 맞게 내용만 재정의.
### 4-3. abstract-class (추상 클래스 & 인터페이스)
  - **추상 클래스**: 순수 가상 함수를 포함하여 직접 객체 생성이 불가능한 '설계 전용' 클래스.
  - **인터페이스(Interface)**: 구현부 없이 규격만 정의하여 자식 클래스에게 특정 함수 구현을 강제함.
  - **표준화**: 서로 다른 클래스라도 동일한 인터페이스를 상속받으면 일관된 명령어(함수)로 제어 가능.
  - **확장성**: 새로운 자식 클래스가 추가되어도 기존의 제어 로직(main 등)을 수정할 필요가 없음.
### 4-4. operator-overloading (연산자 오버로딩)**
  - 객체도 기본 자료형처럼 `+`, `==`, `++` 연산을 할 수 있도록 '단축키'를 설정하는 기법 학습
  - `friend` 키워드와 `ostream`을 사용하여 `cout << 객체` 형태의 사용자 정의 출력 구현
  - `const &`와 `*this`를 활용해 원본 데이터를 보호하고 연속적인 연산이 가능하도록 설계
### 5-1. template-function (함수 템플릿)**
  - `template <typename T>`를 사용해 자료형에 독립적인 범용 알고리즘 구현법 학습
  - 컴파일 타임에 타입별 함수가 자동 생성되는 **인스턴스화(Instantiation)** 과정 이해
  - **주의사항**: `char*`와 같은 포인터 타입 전달 시, 데이터 값이 아닌 '메모리 주소값'이 비교되는 함정과 해결 전략(템플릿 특수화 및 `std::string` 활용) 정리
### 5-2. vector-container (동적 배열)
- **동적 확장**: 데이터 삽입 시 공간이 부족하면 내부적으로 메모리를 재할당(Reallocation)하여 유연하게 대처함.
- **연속 메모리**: 데이터가 메모리상에 나란히 배치되어 인덱스(`[]`)를 통한 임의 접근이 매우 빠름(O(1)).
- **자원 관리(RAII)**: 객체 소멸 시 할당된 메모리가 자동으로 `delete`되어 메모리 누수를 방지함.
- **최적화 포인트**: `size`(요소 개수)와 `capacity`(공간 크기)의 차이를 이해하고, 대량의 데이터 전달 시 참조형(`const &`)을 사용하여 복사 비용을 절감함.
### 5-3. string-class (표준 문자열)
- **추상화**: C의 `char*`를 객체화하여 메모리 할당/해제, 길이 계산 등을 자동화함.
- **직관적 연산**: `+`(결합), `==`(비교), `+=`(추가) 등 연산자 오버로딩을 통해 가독성 높은 코드 작성 가능.
- **안전한 검색**: `find()`와 `string::npos`를 이용해 문자열 존재 여부를 안전하게 확인.
- **하위 호환성**: `c_str()` 멤버 함수를 통해 기존 C 스타일 API(`const char*` 요구 함수)와의 연동 지원.
- **성능 고려**: 빈번한 문자열 결합(`+`)은 재할당 오버헤드를 유발할 수 있으므로 상황에 따라 `reserve()` 사용 고려 권장.
### 5-4. iterator (반복자)
- **개념**: 컨테이너 원소를 가리키는 '손가락(포인터)'. 모든 STL에 공통 적용됨.
- **범위**: `begin()`(첫 원소)부터 `end()`(**마지막 다음 경계**)까지 순회.
- **연산**: `*it`으로 값을 꺼내고, `++it`으로 다음 칸 이동, `!=`로 종료 판단.
- **실무**: 타입명이 길어 보통 `auto`를 사용하며, 역순 순회 시 `rbegin/rend` 활용.
- **주의**: 벡터의 크기가 변하면(재할당) 기존 반복자는 '무효화'되어 사용 불가함.
